# x402 V2 Protocol Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate RustyClawRouter from x402 V1 wire format to V2 spec, enabling standard interop with the x402 ecosystem (facilitators, SDKs, dashboards).

**Architecture:** V2 changes are mostly structural — the same data moves to different locations in the JSON and different HTTP headers. The core verification logic (Solana tx introspection, escrow deposit/claim) is unchanged. We update the wire format types in `protocol`, adjust the gateway's 402 response emission and success response headers, update the `/v1/supported` endpoint to V2 format, and update all 4 SDKs. **Backward compatibility strategy:** During the transition, the gateway emits BOTH the `PAYMENT-REQUIRED` header (V2) AND the full `PaymentRequired` JSON body (V1) on 402 responses. This ensures existing deployed SDKs continue to work. A future task removes the body once all SDKs are updated and deployed. Incoming `PaymentPayload` deserialization accepts both V1 (with `method`, without `extensions`) and V2 shapes via `#[serde(default)]`.

**Tech Stack:** Rust (Axum), serde, base64, Python, TypeScript, Go

---

## Scope & Breaking Changes

| V2 Change | Current State | Impact |
|-----------|---------------|--------|
| 402 body → `PAYMENT-REQUIRED` header (base64 JSON) | 402 returns JSON body via `GatewayError::InvalidPayment` | **Breaking** — gateway + all SDKs |
| Add `PAYMENT-RESPONSE` header on success | Not implemented | **New** — gateway middleware |
| `Resource` gains `description`, `mimeType`; drops `method` | `Resource { url, method }` | **Breaking** — protocol types |
| `PaymentPayload` adds `extensions` field | Not present | **Additive** |
| `PaymentRequired` adds `extensions` field | Not present | **Additive** |
| `SettlementResult` adds `amount`, `extensions` fields | Has `verified_amount` | **Additive** |
| `/supported` → V2 format with `signers` map | Custom `SupportedResponse` | **Breaking** — endpoint schema |
| `cost_breakdown` is RCR-specific (not in V2 spec) | In `PaymentRequired` body | Move to `extensions` |

## What Does NOT Change

- Solana transaction verification logic (`crates/x402/src/solana.rs`)
- Escrow verification logic (`crates/x402/src/escrow/verifier.rs`)
- Escrow claim pipeline (claim queue, claim processor, claimer)
- Fee payer pool, nonce pool
- Smart router, provider adapters, caching, usage tracking
- `PAYMENT-SIGNATURE` header name (already V2)
- CAIP-2 network identifiers (already V2)
- `x402_version: 2` constant (already V2)

```rust
GatewayError::PaymentRequiredV2(pr) => {
    let json_string = serde_json::to_string(&*pr).unwrap_or_default();
    let encoded = base64::engine::general_purpose::STANDARD.encode(json_string.as_bytes());

    // TRANSITION PERIOD: emit BOTH the V2 header AND the V1 body.
    // Existing SDKs/CLI parse PaymentRequired from the response body.
    // New V2 clients read the PAYMENT-REQUIRED header.
    // Once all SDKs are updated and deployed, a future task can change
    // the body to `{}` and remove this dual-emission.
    let body_json: serde_json::Value = serde_json::from_str(&json_string)
        .unwrap_or(json!({}));
    let mut response = (
        StatusCode::PAYMENT_REQUIRED,
        axum::Json(body_json),
    ).into_response();
    if let Ok(hv) = HeaderValue::from_str(&encoded) {
        response.headers_mut().insert(
            HeaderName::from_static("payment-required"),
            hv,
        );
    }
    return response;
}
```
